feature: **Inline text spans in Native markup**: `<text>` now takes inline `<span>` children — mixed-weight (`weight="regular|medium|bold"`), mono, italic, and token-colored (`foreground`) runs word-wrap as one paragraph, `{bindings}` interpolate inside spans, and both markup engines lower them through the same span-paragraph builder Zig views use, so a markup paragraph renders and announces (as one text run) exactly like a builder one.
- **Whitespace between inline runs is structure**: the parser collapses source whitespace between a span paragraph's runs to a single spliced space (none means the runs abut), so spacing survives NSUI round-trips and document hashing instead of living in formatting.
- **Span teaching errors**: spans outside `<text>`, spans in single-style labels, unknown span attributes, unknown weight values, nested spans, empty spans, and dead `wrap`/`overflow` on an always-wrapping span paragraph all fail with pinned messages in the validator and both engines.
