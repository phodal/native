feature: **Disclosure trees with the ARIA tree keymap**: a `tree` container element (markup `<tree>`, Zig `ui.tree`) whose descendant rows carrying `role="treeitem"` — at ANY nesting depth — form one roving keyboard focus set: Up/Down walk the visible rows in tree order, Left collapses an expanded row or moves to the parent row, Right expands a collapsed row or moves to the first child row, Home/End jump to the scope's edges, and Enter/Space activate. Selection FOLLOWS focus the TEA way — every focus arrival dispatches the landed row's `on_press` Msg, so the model owns selection exactly like a click — and disclosure stays model-owned too: expandable rows bind the new `expanded` attribute plus `on-toggle`, and collapsed subtrees are simply not rendered.
- **Routed keyboard events now carry `focus_moved`**, letting tree rows (and future composites) tell "focus arrived here" from "an arrow landed in place" — the collapse/expand intents fire only in place, so Left-to-parent never collapses the parent it lands on.
- **Tree selection is single-select across the whole tree scope** (not per nesting level), with the engine's selected/expanded echoes riding the existing optimistic-echo contract: the model's rebuild is truth.
- **The notes folder rail is now a tree**: Up/Down/Home/End move the folder selection and Enter activates, through the same rows the mouse clicks.
- New `tree`/`treeitem`/`separator` accessibility roles across snapshots and the platform bridges (exposed as list/listitem/group where the native ABI predates them), both markup engines in parity, LSP docs/completions, and skill + docs coverage.
