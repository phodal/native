fix: **Signed packages keep a valid code signature**: the packager wrote its signing-plan record into `Contents/Resources` after codesign ran, breaking the resource seal — every signed bundle failed `codesign --verify --strict` and a quarantined install hit Gatekeeper's "damaged" dialog. The plan is now sealed into the bundle before signing (only a failed codesign rewrites it), codesign failures are detected instead of ignored, and `zig build test-package-signing` pins strict verification in CI.
