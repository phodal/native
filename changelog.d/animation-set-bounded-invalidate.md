fix: **Setting render animations no longer invalidates the whole view**: `setCanvasRenderAnimations` marked the full view frame dirty on every call — and a `UiApp` with an `animations` hook re-bases its set on every rebuild, so each update silently defeated incremental presentation (the region usually fell off the 8-slot dirty buffer, which is why nothing failed). Scheduling now invalidates the union of the targeted commands' display-list bounds widened by each animation's from/to transforms (rotations by their circumscribed square), replacing an empty set with an empty set is a no-op, and unknown targets stay loud with the full view.
