feature: **Display-cadence frame pacing and runtime layout tweens**: an armed frame channel now delivers exactly one frame event per display interval. The macOS host previously ran two independent once-per-interval emission streams (runtime frame requests and GPU present completions) against one shared pacing clock, so an armed animation loop whose phases interleaved delivered two events per interval (measured 3.7 ms/5.5 ms alternation on a 120 Hz panel — double the re-render work), and under sustained main-thread load the late-fire path re-based the clock from completion time each cycle, stretching armed intervals frame over frame. Every producer now funnels through one per-surface scheduler that coalesces into a single paced emission and advances its clock along the display-interval grid — steady 8.3 ms cadence at 120 Hz, unarmed apps still present zero frames. On top of the fixed cadence, layout animations become a runtime primitive: `UiApp.Options.layout_tweens` (or `runtime.startCanvasWidgetLayoutTween`) declares a split's target fraction and the runtime eases the rendered fraction toward it, one step per presented frame — each step lands through the divider-drag mutation path, so retained-diff dirty regions, `on_resize` echoes, and the source-wins reconcile all behave exactly as a drag, reduced-motion appearances snap, and steps sample only recorded frame timestamps so session replay reproduces identical frames. The `examples/split-collapse` harness demonstrates both modes and logs real frame deltas during its 180 ms collapse.
