fix: **Enter edits in a textarea**: a multi-line editor treats plain Enter (and Shift+Enter) as a newline insert instead of swallowing it — the honest layer split: single-line kinds keep enter-to-submit, and a textarea with `on-submit` dispatches on the primary chord (cmd+Enter on macOS, ctrl+Enter elsewhere). The mapping is one shared function (`widgetKeyboardNewlineTextEditEvent`) consulted by both the runtime edit path and the app `on_input` dispatch, which also fixes a silent model desync: the newline previously landed only in the retained widget text and never reached the model's `on_input`.
