improvement: **Mobile frames raster and upload only what changed**: the embed pixel ABI gains `native_sdk_app_render_pixels_damage` — an incremental sibling of `native_sdk_app_render_pixels` for hosts that retain their buffer — which copies only the region the frames since the previous call changed and reports it in device pixels (empty damage = skip the upload). Behind it, the embedded runtime's pixel presents now plan incrementally: packet presenters are dropped from the mobile services (their presence forced every changed frame to a full repaint), pixel presents keep the keyed command mirror alive so a keystroke's dirty bounds refine to the field instead of the window, and the dirty-scissored CPU raster repaints just that region. The iOS host uploads only the damaged rect (`replaceRegion` with the rect; a no-damage revision bump vends no drawable) and the Android host locks/posts only the dirty rect; both keep the full-upload fallback when damage is absent. The reference renderer additionally decodes sRGB through an exact 256-entry table and caches scale-once sample panels for integer-aligned image draws (memo-attached surfaces), so an album grid of photographic covers rasters in a fraction of its former full-surface cost — bytes identical everywhere, only time moves.
