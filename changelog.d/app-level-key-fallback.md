feature: **App-level key fallback (`UiApp.Options.on_key`)**: a key_down that widget routing declines now reaches the app — nothing focused, or the focused widget neither binds the key nor consumes it structurally — with a pinned precedence rule: the focused widget's own activation wins, editable text widgets block fall-through by KIND (typing stays typing even with no `on_input` bound), and only then does the app hook see the key. The honest home for unmodified media keys, which chrome shortcuts deliberately refuse.
- **Space toggles transport app-wide in the music examples**: soundboard and deck adopt the media-app convention through `on_key` — space plays/pauses from anywhere, a focused track row still plays that row, and the search fields keep typing spaces; both suites pin the rule through the raw key path.
