feature: **The stock theme follows the system appearance**: an app that sets neither `tokens` nor `tokens_fn` derives its tokens from the OS light/dark setting (plus high-contrast and reduce-motion) and re-themes LIVE when the user flips it — no restart, no wiring. Explicit `tokens` (fixed look) and `tokens_fn` (model-owned theming) opt out unchanged; test/null platforms never emit appearance events, so deterministic runs stay on the default light theme.
