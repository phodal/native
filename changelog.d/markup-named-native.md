improvement: **The markup language is Native markup, extension `.native`**: the declarative view format formerly named ZML is renamed — views live in `.native` files (`native init` scaffolds `src/app.native`), the docs, skills, examples, evals, and the editor extension (`editors/native-markup/`, language id `native-markup`, matching both extensions) all teach the new name. Existing `.zml` files keep working everywhere for now — loaders, imports (which resolve by the path as written, so mixed `.zml`/`.native` closures are fine), hot reload, `native markup check`, and `native check` all accept them — and the checker prints a per-file note ("rename to .native — .zml keeps working for now"); the `.zml` alias will be removed in a later release once downstream apps have renamed.
