feature: **Resizable split panes**: a first-class `split` container (markup `<split value="{fraction}" on-resize="msg">`, Zig `ui.split(.{ .value = f, .on_resize = Ui.valueMsg(.tag) }, .{ pane1, pane2 })`) with a synthesized draggable divider between exactly two panes — resize cursor, focusable handle with ARIA separator semantics (Left/Right step the fraction, Shift for 2x, Home/End jump to the clamp edges), and `min-width` on the panes bounding the drag. The fraction is MODEL-OWNED: every drag/keyboard/assistive resize dispatches a `canvas_widget_resize` event that `UiApp` maps to the split's `on_resize` Msg with the applied fraction, and echoing it back through `value` is the controlled loop; an unbound split keeps its divider position across rebuilds under the same source-wins reconcile rule as scroll offsets (the restore re-runs the split's child layout in place, so panes land exactly where the divider sat).
- **Automation and snapshots**: the divider appears as `role=separator` carrying the fraction as its value with drag/increment/decrement actions, `widget-drag` drives it through the real pointer pipeline, and `widget-key` reaches the keyboard steps. Pending resizes drain like scroll events, deduped per split (`max_canvas_widget_resize_events_per_view`).
- **The notes example three-pane frame is now two nested splits** (sidebar seam + list/editor seam) with model-owned fractions replacing the fixed 216/304 column widths.
- Both markup engines in parity (validator, interpreter, comptime compile errors — `split` takes exactly two element children, `on-resize` is split-only and names an f32 Msg variant), LSP docs/completions, and skill + docs coverage.
