feature: **The model–view contract, checked in both directions at check time**: `zig build model-contract` (every scaffolded app has it; `zig build test` refreshes it) reflects the app's `Model`/`Msg` into `zig-out/model-contract.zon`, and `native check` verifies every binding path, `for each` iterable, `key` field, message tag, payload type, and typed expression in every `.native` (through the whole import closure, template args flowing use-site kinds into template bodies) against that surface — in milliseconds, without compiling the app, with did-you-mean over the model's actual names and expression type errors naming the field and its Zig type.
- **Dead-state lint**: model fields, query fns, and `Msg` tags no view binds or dispatches are warnings (update-only state opts out with `pub const view_unbound = .{ "name" };` on `Model` or `Msg`; `--strict` promotes warnings to failures).
- **Never a false pass**: the artifact carries a hash of the app's Zig sources, and a missing, stale, or unreadable artifact degrades `native check` to structural checking with a note; a conformance suite holds the contract checker and the binding engines to identical accept/reject sets with the same teaching messages.
