feature: **Quiet-surface hover knob (`WidgetStyle.quiet_hover`)**: hover washes belong to acting controls — list rows, menu items, buttons, tab triggers — where the fill is the affordance; on an image-forward content tile (a cover-art grid, a photo card) the pointer rests on content, not a control register, and a wash reads as a smudge. `.style = .{ .quiet_hover = true }` states that per widget and silences ONLY the hover fill: the pressed wash still marks the moment of commitment, and the focus ring, cursor intent, and hit testing keep their own channels. Every state-washing fill resolver honors the knob through one shared read (`washHovered`); a menu row's keyboard attention wash deliberately stays, because inside a menu that wash is the keyboard affordance. Zig-side style vocabulary only — deliberately no new markup attribute. The soundboard's album tiles adopt it, replacing the hand-rolled transparent-background override (which also killed press feedback and any base fill the tile might want); the doctrine and the knob are documented on the building-components page.
