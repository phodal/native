feature: **Launch-to-glass lap channel**: `NATIVE_SDK_WINDOW_TIMING=1` now prints a wall-clock lap line (`native-sdk: launch <phase> wall_ns=`) at each startup phase boundary — runner entry, AppKit/NSApp init, font registration, menu build, app configuration, window chrome, window create, runtime init, app start, scene load, first view build, and first packet present — and the existing window-shown line carries a matching `wall_ns=` stamp, so a harness that wall-clocks the process spawn can attribute the entire launch, exec to glass, from one env var. Zero-cost when unset: one cached env read gates every lap.
- **Frame-interval profile stage**: while `automate profile on` is active, the frame profile records the measured gap between consecutive gpu-surface completion events as an `interval` stage (`interval_p50_us=`/`interval_p90_us=`/`interval_max_us=` on the snapshot's `frame_profile` line) — the refresh-hold verification channel: a paced loop holding 120 Hz reads p50 ≈ 8333, and a dropped frame shows as max well above p50.
