feature: **Chart axis labels and hover details**: charts grow reference-grade reading aids, all opt-in. `x-labels` binds a model iterable of strings (one category label per sample, drawn muted below the plot and deterministically thinned to fit; dropped when a series downsamples), `y-labels` adds numeric ticks on a nice-step lattice (1/2/5 × 10^k — every label exact at its precision, and gridlines ride the same lattice so grid and labels never disagree), and `hover-details` snaps the pointer to the nearest sample and floats a detail card (sample label, per-series swatch/name/value, hairline cursor, dots on line points) that repaints only when the snapped sample changes. Both markup engines, the validator, the contract checker, NSUI, and `ui.chart` carry the new options (attr codes 73–75); static renders stay byte-identical — hover chrome exists only under live interaction, and unlabeled charts render exactly as before.
