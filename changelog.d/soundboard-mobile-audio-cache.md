fix: **Soundboard streams cache on phones**: the mobile entry (`initModel`) now resolves the platform cache directory and the `NATIVE_SDK_MUSIC_URL_BASE` override once at boot, exactly like the desktop launch path — previously the embed entry seeded neither, so `fx.playAudio` carried an empty `cache_path` on iOS and Android and every play of a streamed track re-downloaded it. The embed entry has no `std.process.Init`, so the environment is read through libc (both phone hosts publish `HOME`/`TMPDIR` before app code runs); the mobile-entry pin now covers the resolved cache directory.
