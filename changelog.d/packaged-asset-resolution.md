fix: **Packaged macOS apps can actually read their bundled assets**: packaging flattened `assets/` into the `Contents/Resources` root while the runtime resolved relative asset paths against the process cwd, so a bundle launched from Finder carried its assets as dead bytes it could never open. The bundle now mirrors the asset tree at its app-relative path (`Contents/Resources/assets/`) and the macOS host resolves relative asset file paths (audio sources) against `Resources` inside a packaged app — the same contract webview asset roots and bundled fonts already follow — so `assets/music/track.mp3` plays from the bundle exactly like it does in a dev run.
