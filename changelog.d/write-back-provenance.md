feature: **Two-way tooling — widget provenance and markup write-back**: select a widget in a running app, jump to the markup that authored it, and write the edit back into the source file, with the app's hot-reload watch closing the loop.
- **`native automate provenance <view-label> (<widget-id> | at <x> <y>)`**: reports where a live widget was authored — file, byte span, line:column, the template instantiation chain (definition site plus every `<use>` that put it there), and the `<for>` iteration keys. Zig-builder widgets answer `authored=zig` honestly.
- **`native automate edit <view-label> <widget-id> <set-attr|remove-attr|set-text> ...`**: applies one minimal-diff operation to the markup file backing the widget (imported component files included) — only bytes inside the target node's span change, proven by reparse plus a parse-tree diff before anything is written.
- **Refusals never touch the file**: a Zig-authored widget, a file that changed on disk since the app loaded it (per-file content hashes guard concurrent edits), or an edit that would fail markup or whole-closure validation each stop with a teaching error.
- **Provenance capture is flag-gated and budgeted**: the markup engines stamp sources only when the builder carries a sink (automation-enabled, markup-interpreter builds), records live in a fixed per-view pool (`canvas_limits.max_canvas_widget_provenance_records_per_view`), and the pool commits its file anchors only on a successful reload adopt.
- **`ui_markup.edit`**: the std-only minimal-diff op layer (`applyChecked`: parse, span-local apply, reparse, validate, structure-preservation diff) is a public canvas module for tools beyond the CLI.
- **`zig build test-writeback-smoke`** (macOS, also in `scripts/gate.sh full`): drives the whole loop against the kanban example — provenance query, label flip through the verb, repaint assert, byte-exact diff check, and flip-back restore.
