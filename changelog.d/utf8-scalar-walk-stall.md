fix: **One stray invalid UTF-8 byte could hang the renderer**: `nextTextOffset` on an orphan continuation byte (0x80-0xBF with no lead) snapped the cursor back to the previous scalar's lead and returned an offset at-or-before its input, so every scalar loop built on it — glyph-atlas planning, text-layout line wrap, caret movement — spun forever the moment such a byte reached a rendered string (found by the hostile-markdown corpus: a document with `\x80` in a paragraph froze frame planning). Invalid bytes now always advance the walk by one byte (the fallback-scalar rule the measure/render paths already follow); pinned by an advance-invariant test over orphan continuations, truncated leads, surrogate halves, and NULs.
