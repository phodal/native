feature: **Icon-in-button (`<button icon="save">Save</button>` / `ElementOptions.icon`)**: buttons draw a vector icon inline before the label — or centered alone when the label is empty — as part of the button's OWN rendering, so icon + label are one widget, one hit target, one tint that follows the button's enabled/disabled/variant state. This retires the overlay idiom's two costs (hand-tracked tint on the stacked icon, duplicated `on_press` on text overlays); markup validates the name against the closed icon vocabulary (comptime in the compiled engine, teaching errors in the interpreter/validator/LSP, button-scoped), `toggle_button` inherits the channel in Zig views, and intrinsic button sizing accounts for the icon through shared metrics so measured width matches painted pixels.
- **`icon_button` draws real vector icons**: a registry (or app-registered) name in `widget.icon` or `widget.text` renders as tinted paths through the shared icon emitter; non-registry text keeps the historical glyph rendering, so existing glyph icon buttons are untouched.
- **Showcase retrofits**: the soundboard transport is real `skip-back`/`pause`·`play`/`skip-forward` icon buttons (chevron stand-ins gone) and its Back/Play-album/clear-search overlays collapsed to single buttons; the markdown-viewer toolbar gained `folder-open`/`save` icons whose disabled tint finally matches the label plus a sun/moon icon-only theme toggle; notes folder rows wear the new `folder` icon and its header uses icon-only clear/theme toggles and a `plus` on New note.
