improvement: **The macOS packet host stops re-rasterizing what didn't change**: steady-state present cost on a live animated window dropped an order of magnitude (host draw p50 from milliseconds to ~0.3 ms on the gpu-dashboard pulse) through four host-side changes that alter zero pixels. (1) A per-command raster cache: each retained command's painted output (text runs, shadows, shapes — not backdrop-reading blurs) is rasterized once at backing scale and blitted thereafter, keyed by the engine's retain key, validated by command identity, evicted with the retained list (patch upserts/evicts, baseline rebuilds, scale/size changes), and byte-budgeted with LRU eviction — CoreText shaping and glyph rendering no longer run per frame for unchanged text. (2) The patch scissor snaps outward to the device-pixel grid, so region-boundary pixels are repainted whole instead of antialiased against retained history — no seams, and incremental output is byte-identical to a full redraw. (3) Full-surface passes paint the retained backing in place (validity-flagged against mid-draw failure) instead of allocating a fresh 16 MB buffer and copying it back after upload. (4) Present-completion frame events are paced to the display's refresh interval and coalesced — the unpaced loop span the engine's frame cascade at ~240 Hz and then stalled multiple milliseconds per present waiting on the exhausted drawable pool. Fidelity is enforced, not assumed: `NATIVE_SDK_GPU_VERIFY_INCREMENTAL=1` makes the host byte-compare every scissored dirty update against a from-scratch full redraw of the same retained list, and the gpu-dashboard smoke drives clicks, the pulse animation, and a mid-animation resize under it, failing on any mismatch. `NATIVE_SDK_GPU_DRAW_TRACE=1` prints a per-present draw/present phase split for diagnosis.
