feature: **Geist Mono is bundled — mono runs ink a real fixed-pitch face on every headless text path.** The reference renderer (the CPU path behind `automate screenshot`, `render_pixels`, mobile embeds, and every pixel golden) previously borrowed the proportional sans outlines for mono font ids, centered inside the fixed 0.6 em cell — an honest stopgap that still read as broken letterspacing at caption sizes: wide caps (M, W: ~0.83 em of ink) overflowed the cell into their neighbors ("Mbdels", "SPECTRUMV/32") while narrow glyphs floated in gulfs ("L I VE"). `canvas.font_ttf` now embeds and comptime-validates GeistMono-Regular (OFL, same family as the bundled sans) and the reference renderer routes the reserved mono id to it. Layout metrics are unchanged by construction: every covered Geist Mono glyph advances exactly the 600/1000 units the deterministic estimator has always charged for mono runs, and a comptime check holds the bundled pitch to the estimator's constant so a future bundle swap cannot drift ink from measurement. Sans-only frames are byte-identical; the dev2-README markdown signature regenerated (2.66% of pixels, all inside mono runs, spot-reviewed for overlap/clipping). Live macOS windows keep drawing mono through the system text stack, unchanged. The pitch is exported as `canvas.mono_advance_em` so apps that snap mono sizes to whole-pixel pitches (small engraved captions live and die by advance phase) derive from the same constant the engine measures and inks with.
