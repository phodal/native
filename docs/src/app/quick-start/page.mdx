# Quick Start

Native SDK is the complete toolkit for building native desktop and mobile applications — with selectable web engines, native views, guarded bridge commands, and OS capabilities. Use a WebView for rich product UI, then add native shell surfaces where the operating system should own the interaction.

## Beta scope

System WebView builds are available on macOS, Linux, and Windows. Chromium/CEF builds and packages are currently supported on macOS only. Native controls are supported on macOS, Linux, and Windows system-WebView hosts.

## Prerequisites

- [Zig 0.16.0+](https://ziglang.org/download/)
- Node.js with npm for the generated frontend
- macOS 11 or newer, Linux, or Windows

## Track 1: Simple WebView App

```bash
native init my_app --frontend next
cd my_app
```

Frontend options: `next`, `vite`, `react`, `svelte`, `vue`.

## Run it

```bash
zig build run
```

The first frontend build installs dependencies automatically. Your app opens a native window with a WebView rendering your HTML.

## Track 2: Native Shell App

Use the native-shell example when you want native chrome around WebView content:

```bash
cd examples/native-shell
zig build test
zig build -Dplatform=macos -Dweb-engine=system
```

The example builds a native toolbar, titlebar accessory, sidebar, statusbar, and WebView content area. Commands can come from native controls, menus, shortcuts, or the WebView bridge.

## macOS package path

Use this path when validating a signed macOS app:

```bash
native init my_app --frontend next
cd my_app
zig build run
native cef install
zig build run
native package --target macos --signing identity --identity "Developer ID Application: Your Name"
native doctor --manifest app.zon --strict
```

Set `.web_engine = "chromium"` and `.cef = .{ .dir = "third_party/cef/macos", .auto_install = false }` in `app.zon` before the Chromium run. `-Dweb-engine` and `--web-engine` are still available for one-off overrides, but the normal app workflow reads the manifest.

For frontend frameworks, run the frontend dev server through [Dev Server](/cli/dev) during development, then package the built assets for distribution.

## Hello world

The simplest Native SDK app provides a name and inline HTML:

```zig
const HelloApp = struct {
    fn app(self: *@This()) native_sdk.App {
        return .{
            .context = self,
            .name = "hello",
            .source = native_sdk.WebViewSource.html(
                \\<!doctype html>
                \\<html>
                \\<body style="font-family: system-ui; padding: 2rem;">
                \\  <h1>Hello from Native SDK</h1>
                \\</body>
                \\</html>
            ),
        };
    }
};
```
