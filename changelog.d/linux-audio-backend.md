feature: **Linux audio playback and streaming**: the GTK host gains the full audio effect family through a GStreamer playbin pipeline, matching the macOS and Windows contract exactly: local MP3 playback, progressive HTTP(S) URL streaming with honest `buffering` reports (including the mid-stream refill discipline), pause/resume, accurate seek, per-player volume, ~500ms position ticks only while playing, and one completion at natural end. URL sources resolve verified-cache-first, then stream while a parallel libsoup download fills the shared track cache (part file, size-verified against the manifest, atomic rename — a partial file never occupies the cache name), so the second play of a track is local. GStreamer and libsoup are loaded at runtime like libsecret, keeping the toolkit's build inputs unchanged: on a host with GStreamer installed, `audio_playback` and `audio_streaming` report supported on the Linux system-WebView host and the soundboard and deck examples play for real; without it the capabilities honestly answer false and playback degrades to one explicit failed Msg — never a crash.
