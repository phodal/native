feature: **Button groups render as one segmented bar**: a `button_group` at its default gap of 0 stamps each visible child's position into the render walk (both the retained tree walk and the layout walk), so the first segment keeps its leading corners, the last its trailing corners, middles square off, and every non-first segment drops its left border — each interior boundary is exactly ONE shared 1px seam (two overlapping translucent dark-scheme hairlines would composite into a brighter line than the group's outer edge). Grouped variants keep their fill with the same seam collapse; a gap above 0 opts back into ordinary separate buttons. The component gets its own docs page with an engine-rendered hero, live previews, and markup examples.
