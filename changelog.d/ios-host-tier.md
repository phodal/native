feature: **iOS graduates to a host tier — the toolkit owns the entire iOS app**: a complete UIKit host (src/platform/ios/uikit_host.m — full-screen canvas surface, touch/keyboard/IME forwarding, CoreText text measurement, safe-area reporting) now ships in the SDK and is built on the embed C ABI; an app project carries zero host code, and embedding a hand-written host stays first-class.
- **`native dev --target ios [--device name]`**: builds the app's embed static library for the simulator (the `lib` step `addApp` now registers for mobile targets, so zero-config apps work with nothing but `-Dtarget`), wraps it in the toolkit host, installs and launches it via `simctl`, and streams the app log. Markup hot reload does not reach the simulator yet — edit, then rerun.
- **`native package --target ios` emits an archive-ready Xcode project**: a deterministic project.pbxproj + shared scheme generated at package time (fixed object ids, byte-identical runs), the generated Info.plist, the asset catalog rendered from the single-source icon pipeline, bundled app assets, and a freshly built device-slice library — `xcodebuild archive` works with zero edits; code signing stays manual, like macOS notarization. iOS bundle ids normalize underscores to hyphens (Apple's identifier grammar), mirroring the Android mapping.
- **Safe areas ride the window-chrome channel**: mobile embed hosts publish the viewport's safe-area insets through the same `on_chrome` channel macOS answers with its titlebar band, so apps pad with one code path everywhere. Subscribing transfers safe-area padding ownership to the app (the runtime keeps only the keyboard's residual inset); unsubscribed apps keep the automatic runtime inset unchanged.
