improvement: **The first canvas frame presents immediately instead of waiting for the placeholder timer**: a canvas view's first frame request used to be dropped while no canvas texture existed, leaving first paint to the 60 Hz placeholder loop — a measured ~40 ms of launch-to-glass latency, plus another ~40 ms when the request's main-queue hop waited for `[NSApp run]` to start pumping. The host now emits the first frame request directly (one-shot; there is no drawable pool to protect before anything has ever presented) and flushes a request queued during the START dispatch synchronously before entering the run loop, so first content hits the glass before the event loop even starts. The dock icon's `.icns` decode also moved off the launch path (~25 ms), updating the dock tile a few frames after launch instead.
