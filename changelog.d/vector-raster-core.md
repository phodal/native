feature: **Deterministic vector rasterizer (`canvas.vector`)**: one std-only, allocation-free CPU path core — bezier flattening with device-pixel tolerance, anti-aliased scanline fill (4x vertical subsamples, analytically exact horizontal coverage) under both `nonzero` and `even_odd` rules, and stroke-to-outline with butt/round caps plus miter (with limit, bevel fallback) and round joins. The raster math uses only `+ - * /` and `@sqrt`, so coverage is bit-identical across platforms; round caps/joins and SVG endpoint arcs (`PathBuilder.arcTo`) are lowered to cubic Beziers via the kappa constant and sqrt-only half-angle identities, keeping the `PathElement` wire model, serialization, hashing, and AppKit packet path untouched.
- **Anti-aliased `fill_path`/`stroke_path` in the reference renderer**: path commands now rasterize through the vector core instead of point-sampled containment/distance tests — smooth edges in screenshots, `render_pixels`, and headless frames. Fill keeps the historical even-odd interiorness; strokes keep round caps/joins. Two stroke-cap pixel goldens in `reference_tests.zig` moved from solid to 75% coverage (the end caps genuinely cover 75% of those pixels); all other reference/example goldens are unchanged.
