fix: **Frames keep pumping during live resize and menu tracking**: the host's frame pump, per-view display timer, automation poll, and app fx timers were scheduled in the default runloop mode only, so AppKit tracking runloops (live window resize, held-open menus) stalled them — content re-laid out only when the gesture paused, and an app's repeating timers (a live sampler, a debounce) froze while a menu was open. All four timers now ride NSRunLoopCommonModes, so resize tracks the drag frame-by-frame and fx timers fire through tracking gestures.
