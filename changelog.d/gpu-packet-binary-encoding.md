feature: **Compact binary GPU packet encoding kills the silent render-path oscillation**: text-heavy frames (a long chat transcript, a dense table) whose JSON packet crossed the 128 KiB transport bound used to fall back SILENTLY to the CPU pixel path every repaint — text visibly flipping between system-text and bundled-face rasterization, shapes going chunky, scrolling turning sluggish. Packets now ride a length-prefixed little-endian binary encoding (`CanvasGpuPacket.writeBinary`, decoded by the AppKit host next to the JSON path) that measures ~10x smaller on a 200-message transcript frame (757 KB JSON → 72 KB binary), under a deliberately raised 512 KiB binary ceiling — the effective packet capacity grew ~40x. Capability negotiation is per-present: platforms that wire the binary presenter get binary, a refused binary call retries JSON in the same frame, and JSON stays for compatibility and wire debugging. Glyph arrays never ride the binary wire (the host draws text from the run's UTF-8 plus the engine-measured lines), which is most of the size win.
- **Fallback telemetry is loud now**: every packet-attempt failure records WHY on the view — `json_overflow`/`binary_overflow` with needed-vs-limit bytes, `unsupported_command` with the offending command kind, or `missing_service` — surfaced in automation snapshots as `present_fallback=… present_fallback_needed=… present_fallback_limit=… present_fallback_command=… present_fallback_frames=…` (the frame counter never resets, so oscillation shows even when a snapshot lands on a healthy frame) and as a rate-limited debug log line.
- **Packet coverage audited**: every display-list draw command kind maps to a representable packet command (structural clip/opacity/transform wrappers flatten into per-command state before the packet), pinned by a kind-coverage test over both encodings; a smoke-level chat-shaped heavy-frame test asserts the packet path holds where JSON alone overflowed.
- **Fallback frames render partially**: the packet→pixel fallback inside `presentNextCanvasFrame` shares one planned frame, so the rare remaining fallback repaints only the dirty bounds instead of a full-frame CPU raster (pinned by test).
