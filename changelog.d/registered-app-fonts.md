feature: **App-registered fonts (`Options.fonts` / `runtime.registerCanvasFont`)**: apps ship their own TrueType faces and register them at startup under an app-owned `FontId` (at or above `canvas.min_registered_font_id`); the id then resolves everywhere a font id rides — `typography.font_id`/`mono_font_id` token overrides, text runs, glyph atlas keys, fingerprints — with the parsed face measuring layout (`canvas.estimateTextWidthForFace`), the reference renderer inking its real outlines, and the macOS host learning the exact face through a new registration side-channel so host measurement and packet text drawing agree. Invalid or hostile font files fail loudly at registration with a teaching diagnosis (`canvas.font_ttf.parseFailureReason`) — never a crash or a silent substitute at render time — and glyphs a registered face does not cover keep the same per-glyph notdef fallback as the built-ins. Budgets are fixed and loud: 8 fonts of 2 MiB each (`canvas_limits.max_registered_canvas_fonts` / `max_registered_canvas_font_bytes`), overflow errors name the budget. The calculator example registers its display face through the new seam.
