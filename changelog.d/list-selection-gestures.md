feature: **Double-click channel**: the typed builder gains `on_double_press` — a release whose click count reaches 2 dispatches it (falling back to `on_press`), so a list row can select on click and play/open on double click; `Ui.Tree.msgForPointerClick` carries the count into typed dispatch.
- **Enter is the row's primary action**: a `list_item` with a bound `on_submit` dispatches it on plain Enter (Space keeps the select activation), the desktop list convention.
- **Quiet list rows are transparent to keys**: quiet focus (what a pointer press or programmatic focus leaves — no ring) on a plain list row no longer claims arrows, Home/End, or activation keys; they fall through to the app's key fallback so an app-owned selection model can drive them, while Tab's ring register keeps the full group-walk and activation behavior. Automation key actions escalate list rows to the ring register so `widget-action <id> press` keeps working.
