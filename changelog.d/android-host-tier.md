feature: **Android graduates to a host tier — the toolkit owns the entire Android app**: a complete Android host (src/platform/android/ — edge-to-edge canvas surface view, touch/soft-keyboard/IME forwarding, Paint text measurement, safe-area and keyboard inset reporting, JNI bridge) now ships in the SDK and is built on the embed C ABI; an app project carries zero host code, and embedding a hand-written host stays first-class.
- **`native dev --target android [--device name]`**: builds the app's embed static library for `aarch64-linux-android`, compiles the toolkit host with the NDK, assembles a debug-signed APK, installs and launches it on a running (or freshly booted) emulator via `adb`, and streams the app log. Markup hot reload does not reach the device yet — edit, then rerun.
- **`native package --target android` emits a complete generated host project and the debug APK**: the toolkit host sources, the app.zon-derived manifest, launcher icons from the single-source icon pipeline, bundled assets, the embed library, and an APK assembled with zero edits directly with the SDK's build tools (aapt2, javac, d8, zipalign, apksigner) — no build-system project, no plugin/version matrix, and dev and package share one assembly so they cannot drift. Store signing keys stay a manual step, like macOS notarization. Application ids keep the hyphen-to-underscore normalization.
- **Soft keyboard and IME are wired**: the host keys the keyboard off the runtime's focus/IME-intent state, routes committed text and composition through the embed IME events, and reports the keyboard's viewport inset so layout clears it — apps that own safe areas via `on_chrome` keep only the keyboard's residual overlap, the same contract iOS established. Display-cutout and system-bar insets ride the window-chrome channel; unsubscribed apps keep the automatic runtime inset unchanged.
