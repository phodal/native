improvement: **The main WebView is created on first use, not at window create**: every window used to instantiate a full WKWebView (and its out-of-process WebKit stack) at create time even when nothing ever loaded into it — for canvas-first apps that was ~33 ms of launch-to-glass latency plus resident WebKit helper processes for a view that sat blank under the canvas forever. Windows now create the WebView lazily when something materializes web content (load, navigate, main-webview frame/zoom/layer placement); peek paths (event emission, bridge completion echoes, reorder passes) skip an absent WebView, whose never-created page has no listeners to miss. File drops used to ride the eager WebView's drag registration, so the window itself now registers for file drops and its delegate (plus any live WebView, which outranks it) forwards them through the same host event.
